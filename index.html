Referências (fontes)

  • Alura - Python: Inteligência artificial aplicada
  • Alura - Python para Data Science
  • Scikit-learn (para o Modelo de Machine Learning): A documentação oficial é a fonte primária para todos os métodos que usamos (como train_test_split, LogisticRegression, .fit(), .predict() e .predict_proba()).
  • Scikit-learn: Machine Learning in Python. (Disponível em: https://scikit-learn.org/stable/documentation.html)

  • Pandas Development Team. (2020). pandas-dev/pandas: Pandas. (Disponível em: https://pandas.pydata.org/docs/)
  • Openpyxl (para o erro de escrita do Excel): O módulo que resolveu o ModuleNotFoundError. A documentação dele explica por que o Pandas precisa dele como "motor" (engine).
  • Documentação Oficial do Openpyxl. (Disponível em: https://openpyxl.readthedocs.io/en/stable/)
  • Microsoft Power BI (para o Dashboard): A documentação da Microsoft é a fonte oficial para o uso da ferramenta, incluindo a lógica de agregação de dados (como a função "Não resumir" que usamos).
  • Documentação do Microsoft Power BI. (Disponível em: https://docs.microsoft.com/pt-br/power-bi/)
  • Conceito de "Churn Rate" (O Tema do Trabalho): Para a definição do problema de negócio (o que é Churn), meu conhecimento é baseado em como esse termo é usado em artigos de ciência de dados e negócios.
  • Artigos em plataformas como Medium (especificamente a publicação "Towards Data Science") ou sites de referência de negócios (como Investopedia ou Harvard Business Review) que definem "Customer Churn Rate".
